# Known Limits and Roadmap

## Documentation Principle

This page exists to stop accidental overstatement. DaisyBase should be sold and operated based on the current shipped surface, not on aspirational features.

## Current Limits Worth Calling Out

- SQL breadth is still narrower than a mature general-purpose commercial RDBMS
- storage and recovery are materially real, but the architecture still has evolution runway
- installer upgrade and uninstall flows are not yet fully automated
- XA and security are real but bounded in enterprise depth
- the PL/SQL bridge is an overlap and translation aid, not blanket PL/SQL compatibility

## Roadmap Direction

- deeper storage/runtime alignment
- broader SQL and planner breadth
- fuller operational tooling
- stronger security and governance depth
- broader installation lifecycle management
