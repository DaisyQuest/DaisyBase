# Code of Conduct

This project values clear technical argument, direct collaboration, and respect for other people's time.

That means:

- disagree with ideas without taking aim at people
- be precise instead of performative
- assume good faith, but not infinite patience
- keep criticism tied to behavior, code, docs, tests, or design

## In Practice

Helpful participation looks like this:

- concise bug reports
- well-scoped pull requests
- honest documentation
- serious review comments with actionable context

Unhelpful participation looks like this:

- personal attacks
- contempt dressed up as expertise
- derailing technical discussion into status games
- vague accusations without evidence

## Enforcement

Maintainers may edit, hide, lock, or remove content that makes the project harder to work on in good faith. They may also block repeat behavior that damages collaboration.

The standard is simple: help make the project more rigorous, more usable, and easier to work on.
