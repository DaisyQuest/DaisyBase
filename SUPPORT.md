# Support

## Where to Start

Before opening an issue, check these:

- [Documentation Handbook](docs/README.md)
- [Installer Guide](installer/README.md)
- [JDBC Module](jdbc/README.md)
- [Demo Business App](demo-business-app/README.md)

Most setup and integration questions are answered there already.

## Good Support Requests

A useful support request includes:

- what you were trying to do
- what you expected to happen
- what actually happened
- the exact command, URL, or configuration involved
- relevant logs, stack traces, or test failures
- operating system and Java version

If JDBC is involved, include the URL shape you used. If the server or installer is involved, include the relevant config file entries.

## What Maintainers Can Help With

Reasonable support topics include:

- build failures
- installer/runtime setup
- JDBC and ORM integration questions
- server startup and configuration issues
- documentation gaps
- behavior that looks incorrect or inconsistent with the docs

## What Makes Support Slow

These patterns waste time:

- "doesn't work" without output
- screenshots of text when the text can be pasted
- missing reproduction steps
- filing a feature request as a bug report

## Feature Requests

Feature requests are welcome when they are concrete.

A good request explains:

- the use case
- why current behavior is insufficient
- what success would look like

## Final Rule

Be precise. A precise rough report is more helpful than a polished vague one.
